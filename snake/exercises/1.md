The most important idea:

**This code builds a 3D scene using Three.js.**
A scene is like a movie stage. You add objects, lights, a camera, then render everything to the screen.

### 1. Import Three.js

```js
import * as THREE from "three";
```

This gives you access to Three.js tools like:

```js
Scene
Camera
Mesh
BoxGeometry
Material
Light
Renderer
```

### 2. Create the 3D world

```js
const scene = new THREE.Scene();
```

The `scene` is the container for everything: car, road, lights, background.

### 3. Create the camera

```js
const camera = new THREE.PerspectiveCamera(60, width / height, 0.1, 1000);
camera.position.set(7, 4.5, 8);
camera.lookAt(0, 0.5, 0);
```

The camera is your eye.

`position.set(7, 4.5, 8)` means:

```js
x = 7      // right
y = 4.5    // up
z = 8      // forward/back
```

`lookAt(0, 0.5, 0)` tells the camera to look toward the car.

### 4. Create the renderer

```js
const renderer = new THREE.WebGLRenderer({ antialias: true });
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);
```

The renderer draws the 3D scene onto the browser screen.

Think:

```js
scene = what exists
camera = what we see from
renderer = draws it
```

### 5. Add lights

```js
const sun = new THREE.DirectionalLight(0xffffff, 2.5);
scene.add(sun);

scene.add(new THREE.AmbientLight(0xffffff, 0.7));
```

`DirectionalLight` is like sunlight.

`AmbientLight` softly lights everything so shadows are not too dark.

### 6. Helper functions make repeated shapes easier

```js
function box(w, h, d, color, x, y, z) { ... }
```

This creates a rectangular block.

Instead of writing long Three.js code every time, you can write:

```js
box(4.8, 0.7, 2.05, 0x1565c0, 0, 0.35, 0)
```

Meaning:

```js
width = 4.8
height = 0.7
depth = 2.05
color = blue
position = x:0, y:0.35, z:0
```

Same idea for:

```js
function cylinder(...)
```

This creates wheels.

### 7. The car is a group

```js
const car = new THREE.Group();
```

A `Group` lets many parts behave like one object.

The car is built from many small pieces:

```js
lowerBody
hood
trunk
cabin
windows
lights
wheels
mirrors
bumpers
```

Then each part is added:

```js
car.add(lowerBody);
car.add(hood);
car.add(cabin);
```

Finally:

```js
scene.add(car);
```

### 8. Wheels are made with cylinders

```js
createWheel(1.45, 1.12);
```

Each wheel has:

```js
tire
rim
hub
spokes
```

The wheel function avoids repeating the same code four times.

### 9. The road is just a flat box

```js
const road = new THREE.Mesh(
  new THREE.BoxGeometry(20, 0.08, 8),
  new THREE.MeshStandardMaterial({ color: 0x2f3542 })
);
```

It looks like a road because it is wide, flat, and dark.

### 10. Animation loop

```js
function animate() {
  requestAnimationFrame(animate);

  car.rotation.y += 0.006;

  renderer.render(scene, camera);
}
```

This runs again and again.

Every frame:

```js
car.rotation.y += 0.006;
```

turns the car slowly.

Then:

```js
renderer.render(scene, camera);
```

draws the new frame.

### 11. Resize handling

```js
window.addEventListener("resize", () => {
  camera.aspect = window.innerWidth / window.innerHeight;
  camera.updateProjectionMatrix();
  renderer.setSize(window.innerWidth, window.innerHeight);
});
```

When the browser size changes, the camera and renderer update so the scene does not look stretched.

### The big mental model

```js
Scene = the world
Camera = your eyes
Renderer = the painter
Mesh = visible object
Geometry = shape
Material = surface/color
Light = makes objects visible
Group = many objects acting as one
Animation loop = redraw again and again
```

This whole code is basically:

**Create a 3D world → build a car from boxes and cylinders → add light and road → rotate the car every frame.**
